## Formato General

### Formato JSON

El formato de salida predeterminado de Aryn DocParse es JSON.

```text
{ "status": in-progress updates,
  "error": any errors encountered,
  "elements": a list of elements }
```

### Formato Markdown

Si la solicitud a Aryn DocParse tiene la `output_format` opción establecida en `markdown`, una respuesta exitosa se verá así:

```text
{ "status": ...,
  "markdown": "# Title\ndolorem ipsum, quia dolor sit amet consectetur..." }
```

## Formato de Elemento

### Estructura General

A menudo es útil procesar diferentes partes de un documento por separado. Por ejemplo, podrías querer procesar las tablas de manera diferente a los párrafos de texto, y típicamente los fragmentos pequeños de texto se incrustan por separado para la búsqueda vectorial. En Aryn DocParse, estos fragmentos se llaman elementos.

Los elementos siguen el siguiente formato:

```text
{"type": type of element (str),
"bbox": Coordinates of bounding box around element (float),
"properties": { "score": confidence score (float),
                "page_number": page number element occurs on (int)},
"text_representation: for elements with associated text (str),
"binary_representation: for Image elements when extract_table_structure is enabled (bytes)}
```

Cada elemento siempre tiene un `type`, `bbox`, `properties`, y `text_representation` campo. El `type` campo indica el tipo del elemento (por ejemplo, texto, imagen, tabla, etc.), el `bbox` campo contiene las coordenadas del cuadro delimitador alrededor del elemento, el `properties` campo contiene información adicional sobre el elemento (por ejemplo, puntuación de confianza, número de página, etc.), y el `text_representation` campo contiene el contenido de texto del elemento. La `score` etiqueta representa la puntuación de confianza en la asignación de etiquetas (`Text`, `Image`, `Table`, etc.). El `page_number` es el número de página indexado en 1 donde aparece el elemento.

A continuación se muestra un ejemplo de elemento:

```json
{
    "type": "Text",
    "bbox": [
      0.10383546717026654,
      0.31373721036044033,
      0.8960905187270221,
      0.39873851429332385
    ],
    "properties": {
      "score": 0.9369918704032898,
      "page_number": 1
    },
    "text_representation": "It is often useful to process different parts of a document separately. For example, you\nmight want to process tables differently than text paragraphs, and typically small chunks\nof text are embedded separately for vector search. In Aryn DocParse, these\nchunks are called elements.\n"
}
```

#### Cuadro Delimitador

```text
"bbox": coordinates of the bounding box around the element contents
```

Toma el formato`[x1, y1, x2, y2]`donde cada coordenada se da como la proporción de qué tan abajo o a través de la pantalla está el elemento. Por ejemplo, un elemento que está a 100 píxeles del borde izquierdo de un documento de 400 píxeles de ancho tendría una coordenada x1 de 0.25.

#### Propiedades

```text
"properties":
    { "score": confidence between 0 and 1, with 1 being the most confident 
                that this element type and bounding box coordinates are correct (float),
      "page_number": 1-indexed page number the element occurs on (int) }
```

La`score`es la "confianza" del modelo en su predicción para ese cuadro delimitador particular. Por defecto, seleccionamos automáticamente los cuadros delimitadores para lograr una buena cobertura con alta precisión de predicción, pero el usuario puede controlar esto usando el`threshold`parámetro (por defecto es "auto"). Si el usuario especifica un valor numérico entre 0 y 1, solo se mantendrán los Elementos con una puntuación de confianza superior al valor umbral especificado.

#### Representación de Texto

```text
"text_representation": text associated with this element (str)
```

Los elementos de texto contienen`\n`cuando el texto incluye un retorno de línea.

#### Representación Binaria

Cuando`extract_images`está configurado como Verdadero, las Imágenes incluyen una`binary_representation`etiqueta que contiene un`base64`codificado`ppm`archivo de imagen del pdf recortado a los límites de la imagen detectada. Cuando`extract_images`es falso, aún se devuelve el cuadro delimitador de la Imagen.

```text
"binary_representation": base64 encoded ppm image file of the pdf cropped to the image (bytes)
```

### Estructura de Tabla

Las tablas se representan como una lista de celdas, donde cada celda es un diccionario con los siguientes atributos:

```json

{
  "type": "table",
  "bbox": [int, int, int, int],
  "properties": {
    "score": float,
    "title": str,
    "columns": int,
    "rows": int,
    "page_number": int
  },
  "text_representation": str,
  "table": {
    "cells": [
      {
        "content": str,
        "rows": list[int],
        "cols": list[int],
        "is_header": bool,
        "bbox": [int, int, int, int],
        "properties": dict
      }
    ]
  }
}
```

Una tabla también contiene una`table`etiqueta con`cells`. Cada celda tiene 6 atributos:`content`,`rows`,`cols`,`is_header`,`bbox`, y`properties`. El`content`atributo contiene el contenido de texto de la celda, el`rows`atributo contiene el índice de fila de la celda, el`cols`atributo contiene el índice de columna de la celda, el`is_header`atributo indica si la celda es una celda de encabezado y es opcional, el`bbox`atributo contiene el cuadro delimitador de la celda, y el`properties`atributo contiene propiedades adicionales de la celda.

La`text_representation`etiqueta contiene el contenido de texto de la tabla y no está especificada o es la representación CSV de la tabla. En el`properties`diccionario, la`title`etiqueta representa el título de la tabla como cadena si se especifica,`columns`es un entero que representa el número de columnas en la tabla, y`rows`es un entero que representa el número de filas. Pueden no estar todos especificados.

### Estructura de Imagen

```json
{ 
  {
    "type": "Image",
    "bbox": [int, int, int, int],
    "properties": {
      "score": float,
      "image_size": [int, int],
      "image_mode": str,
      "page_number": int
    },
    "binary_representation": bytes
  }
}
```

Para una imagen, la`binary_representation`etiqueta contiene un`base64`codificado`ppm`archivo de imagen del pdf recortado a los límites de la imagen detectada. Las propiedades contienen atributos extra:`image_size`, que representa \[`width`,`height`] de la imagen, y`image_mode`, que representa el modo de color de la imagen.
